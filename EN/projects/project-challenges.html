Project Challenges

Seven Seas Cards · Internal Document · Confidential

1. KYC Provider Change — Regulatory Compliance

The original KYC (Know Your Customer) provider was replaced before launch due to updated regulatory requirements.
  • What happened The initially selected KYC provider was found to be non-compliant with new regulatory standards introduced shortly before go-live. A replacement provider had to be sourced, integrated, and tested under significant time pressure.
  • Impact Required rework of the identity verification flow — API integration, document upload, liveness detection, and backend verification pipelines had to be re-tested end to end. Delayed the onboarding module by approximately 2 weeks.
  • Resolution Fast-tracked vendor evaluation; parallel-tracked integration and compliance review. New provider was fully integrated and passed UAT before launch deadline.

2. Currency Exchange API — Rate Update Failure

On the Friday before our weekend, the third-party API responsible for currency exchange rates stopped sending updated rates without warning.
  • What happened The exchange rate API began returning stale/cached rates. FX conversions in the app — card load, balance display, transaction history — showed incorrect amounts for multi-currency cardholders. Detected late Friday afternoon.
  • Impact Risk of cardholders seeing incorrect balances and making financial decisions based on wrong data. Required immediate escalation to the provider and implementation of a fallback mechanism before the weekend.
  • Resolution Implemented a fallback rate source (secondary provider) with automatic failover. Added monitoring alerts for rate staleness (no update in >1 hour triggers notification). Deployed hotfix Friday evening before weekend blackout.

3. Additional Challenges

  • Vendor Coordination Across Time Zones Multiple third-party providers (card processor, KYC, FX, SMS gateway) operating in different time zones created coordination overhead. Issue resolution often delayed by 12–24 hours due to time zone gaps.
  • Regulatory Compliance Timeline Evolving regulatory landscape required mid-project adjustments to compliance features — transaction monitoring thresholds, KYC tier requirements, and reporting formats changed during development.
  • Device Registration & Security Implementing device binding/authorization added complexity to the onboarding flow. Balancing security requirements with user experience required multiple design iterations.
  • Notification System Complexity Push + in-app notifications for 12+ event types (refund, reversal, POS, PIN change, currency exchange, card status, e-commerce, magnetic stripe, etc.) required extensive test case coverage and edge-case handling.
  • Multi-Currency Edge Cases Handling transactions across currencies introduced rounding discrepancies, rate timing issues, and display inconsistencies that required careful business logic and QA validation.
